개발 개요

생성형 AI 영상은 “프롬프트 수정 → 스타일 변경 → 재생성”을 반복하며 결과물을 맞춰가는 경우가 많습니다. 이 과정이 길어질수록 제작 비용(시간)이 커지고, 기대와 다른 결과가 나오면 다시 처음으로 돌아가야 합니다.

AI-Collector는 생성된 영상들을 공유/정리하고, 프롬프트로 빠르게 탐색할 수 있게 만들어 “시행착오를 줄이는 영상 허브”를 지향합니다.

프로젝트 메타

형태 · 1인 풀스택 & DevOps + AI 파이프라인

핵심 · 정리(태그/카테고리) + 추천(프롬프트) + 안전(유해 필터)

정리 추천 안전

문제 정의 & 해결 방향

문제

  • 생성 반복으로 시간이 과도하게 소모
  • 결과물이 기대와 달라 재생성 루프가 발생
  • 참고 자료가 흩어져 있어 비교/탐색이 어려움

해결

  • 생성된 영상을 공유해 레퍼런스를 쌓고
  • 카테고리/태그로 정리하고
  • 프롬프트 기반으로 유사 영상을 추천
  • 유해 영상 필터링으로 안전한 플랫폼 유지

주요 기능

사용자가 “찾고 싶은 영상”에 빠르게 도달하도록, 정리/추천/안전 흐름으로 설계했습니다.

카테고리 정리

  • 태그 기반 필터링, 정렬(최신/조회/반응)
  • 업로드 영상의 프레임/특징 추출(분석용)

프롬프트 기반 추천

  • 목적/스타일/키워드를 입력하면 유사 영상을 추천
  • 결과물 비교 → 프롬프트 수정 비용을 줄이는 UX

AI 유해 영상 필터링

  • AI 1차 감지 → 관리자 검증 → 차단/승인 처리
  • 안전한 공유 환경 유지
