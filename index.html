개발 개요

생성형 AI 영상은 “프롬프트 수정 → 스타일 변경 → 재생성”을 반복하며 결과물을 맞춰가는 경우가 많습니다. 하지만 그 과정이 길고 비효율적이라, 작은 프로젝트 하나를 완성하는 수준의 시간이 들기도 합니다.

AI-Collector는 이런 제작 과정을 “영감 탐색 + 재사용 + 빠른 탐색” 중심으로 바꾸기 위해, 생성된 영상들을 서로 공유하고 프롬프트로 찾아볼 수 있는 허브를 지향합니다.

프로젝트 메타

형태 · 1인 풀스택 & DevOps + AI 파이프라인

목표 · 제작 시행착오 비용(시간)을 줄이고, 원하는 결과물에 더 빠르게 도달

정리/검색 추천/유사도 유해 영상 차단

현재 문제점과 핵심 해결책

문제점

  • 과도한 생성 시간 — 만족할 때까지 반복 생성이 필요
  • 기대 불일치 — 결과물이 기대와 달라 재생성을 유발
  • 참고 자료 부족 — “어떤 프롬프트/스타일이 어떤 결과”를 빠르게 비교하기 어려움

핵심 해결책

  • 공유 — 생성된 영상을 모아 레퍼런스가 쌓이는 구조
  • 정리 — 카테고리/태그/인기 기준으로 빠르게 탐색
  • 추천 — 프롬프트 의도를 분석해 유사 영상 제안
  • 안전 — 유해 영상 자동 감지 + 관리자 검증

주요 기능

“정리 → 추천 → 안전” 3축으로 사용자의 탐색 시간을 줄이는 것을 핵심으로 설계했습니다.

1) 카테고리별 AI 영상 정리

  • 태그 기반 필터링 — 업로드 시 태그를 부여하고, 목록에서 필터링
  • 인기/반응 기반 정렬 — 좋아요/싫어요/조회수/최신순 등으로 분류
  • 영상 특징 분석 — 영상 프레임을 이미지로 변환 후, 특징을 추출해 DB에 저장(분석용)
태그 필터 정렬 특징 분석

2) 프롬프트 기반 영상 추천

사용자가 만들고자 하는 영상의 목적/스타일/핵심 프롬프트를 입력하면, 입력 의도를 분석하고 DB에서 유사 영상을 찾아 추천합니다.

1
프롬프트 입력
목적과 스타일, 핵심 프롬프트를 입력
2
AI 분석
입력 의도 파악 및 특징 추출
3
유사 영상 검색
DB에서 유사 영상 후보군 탐색
4
추천 목록 제공
추천 영상/태그/요약을 함께 표시
추천이 도움이 되는 이유
  • 시행착오를 줄이고, 레퍼런스로 빠르게 방향을 잡을 수 있음
  • 제작 시간을 단축하고, 원하는 결과물에 더 빨리 도달

3) AI 기반 유해 영상 필터링

유해 콘텐츠 가능성이 있는 영상은 자동 감지 → 관리자 검증 흐름으로 운영해 “안전하게 공유할 수 있는 AI 영상 공간”을 유지합니다.

  • 1차 필터링 — AI로 위험 요소 탐지 (예: 성인/폭력 등)
  • 2차 검증 — 관리자 페이지에서 차단 영상 확인
  • 3차 처리 — 차단 해제 또는 삭제
안전 운영 신뢰

※ 구현/연동 방식은 운영 정책과 환경에 맞춰 조정 가능하도록 설계했습니다.

시스템 구조도 (요약)

  • Web · React(Vite) 프론트엔드
  • API · Spring Boot 백엔드(인증/영상/추천/관리자)
  • DB · Oracle (메타데이터/태그/반응/상태)
  • Storage · 영상 파일(경로/메타 분리)
  • AI · 태그/특징 분석 모듈 + 추천 로직
  • Proxy · Nginx + HTTPS(SSL)

구조도 이미지는 PPT/README에 있는 도식으로 교체해 넣으면 완성도가 더 올라갑니다.

아키텍처 설명 (포인트)

  • 정리와 추천은 “탐색 시간 단축”이라는 같은 목표를 공유
  • 필터링(안전)은 운영 신뢰도를 만들기 위한 필수 축
  • 향후 유사도 검색을 강화하기 위해 특징/태그 확장 가능

업그레이드 과정

“정확도 vs 속도” 트레이드오프를 실제로 겪으면서, 태그 방식과 모델 전략을 단계적으로 개선했습니다.

AI Collector v2 — CNN 기반 태그 자동화

  • 사용자가 직접 설정하던 태그를 학습한 CNN 모델로 자동 부여
  • 장점 · 속도 빠름
  • 단점 · 정확성 한계
  • 멀티레이블 특성으로 Softmax 대신 Sigmoid 적용(태그별 독립 확률)
  • 기존 분류 레이어를 제거하고 정의한 태그(예: 14개)에 맞는 출력 레이어를 추가

AI Collector v3 — 비전 LLM(Ollama)로 태그 정확도 강화

  • 태그 정확도를 올리기 위해 비전 LLM 전략으로 전환
  • 속도는 느려지지만 정확도는 개선
현재 방향(확장)
  • 태그/특징 추출 값을 늘려 비교군을 확대
  • 프롬프트 기반 추천 정확도를 높이기 위한 정규화/UX 개선
