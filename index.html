About Project

AI-Collector는 AI·보안·개발·클라우드 관련 영상을 한 곳에 모으고, 자동 태그/요약/검색으로 원하는 영상을 빠르게 찾을 수 있도록 만든 AI 영상 모음 서비스입니다.

YouTube 등 여러 플랫폼에 흩어진 영상을 사용자가 직접 업로드하거나 공유 링크로 등록하면, 프레임 기반 멀티레이블 태깅 모델과 자막·요약 파이프라인(WhisperX + LLM)이 장르·콘텐츠 특성을 분석해 태그·요약·추천에 활용합니다.

이 프로젝트는 졸업 작품 및 생성형 AI 활용 경진대회 출품을 목표로 기획·설계·구현·운영까지 엔드 투 엔드로 진행하고 있습니다.

프로젝트명: AI-Collector · AI 영상 모음 사이트

AI/보안/개발/클라우드 영상을 수집하고, 자동 태그·요약·검색으로 “찾기 편한 AI 영상 허브”를 지향하는 웹 서비스

상태: 구현 & 테스트 운영 역할: 기획 · 아키텍트 · 풀스택 · DevOps · AI

문제 정의

  • AI/개발 영상이 여러 채널과 플랫폼에 분산되어 있음
  • 제목·썸네일·간단한 태그만으로는 난이도/주제/기술 스택 파악이 어려움
  • 비슷한 내용의 중복 영상, 낚시성 콘텐츠, 유해 영상이 섞여 있음
  • 한국어 사용자 입장에서 한글 자막/요약/키워드가 부족함

목표 · “AI 학습자와 개발자를 위한 집중된 영상 허브를 만들자 — 한 곳에서 태그·요약·추천까지 제공하는 서비스”

아키텍처 & 인프라

  • 프런트엔드 · React + Vite, React Router, SCSS
  • 백엔드 · Spring Boot 3, Spring MVC, Spring Security + JWT
  • DB · Oracle 11g XE (별도 노트북, 시퀀스+트리거 기반 PK)
  • 스토리지 · 외장 HDD에 영상 파일 저장, DB에는 경로/메타데이터 저장
  • AI 서비스 · Python 태그 서버(멀티레이블 모델)와 HTTP 연동
  • 인프라 · 두 대의 CentOS 노트북 + TP-Link 스위치로 앱 서버 / DB 서버 분리
  • 배포 · Docker/Compose + Nginx 리버스 프록시 + Let’s Encrypt/Certbot
  • 보안 · iptables로 DB 서버를 앱 서버 IP로만 허용, 80→443 리다이렉트, DKIM 설정

핵심 기능 (사용자)

  1. 회원가입 & 이메일 인증 — 메일 인증 링크를 통한 계정 활성화
  2. 로그인/로그아웃 — JWT 기반 세션 유지, 프론트 Axios 인터셉터에서 토큰 자동 전송
  3. 영상 업로드 — 10초~5분 영상(1080p 기준)을 업로드, 제목/설명/태그 입력
  4. 자동 태그 부여 — 업로드 시 Python 태그 서버를 호출하여 장르/내용 태그 자동 생성
  5. 영상 목록 페이지 — 태그/장르 필터, 최신순/조회순 정렬
  6. 영상 상세 페이지 — 영상 플레이어, 설명, 태그, 자동 태그, 관련 영상 추천
  7. 홈(Story) 영역 — 대표 영상 슬라이더, 사용자가 좋아요/싫어요로 피드백 가능
  8. Finding(찾기) 페이지 — 키워드/프롬프트 입력 기반 검색 UX 설계(벡터 검색 확장 예정)

핵심 기능 (관리자/운영)

  • Admin 전용 로그인 — 일반 사용자와 분리된 JWT, /admin 네임스페이스로 구분
  • 사용자 리스트 — 가입일, 이메일, 토큰 사용량 등 확인
  • 유해/보류 영상 관리
    • 업로드 시 정책 위반 가능성이 있는 영상은 Block 상태로 저장
    • 관리자 대시보드에서 승인 / 보류 / 삭제 처리
  • 로그 확인 — 업로드/에러 로그를 바탕으로 장애 원인 추적

AI 태그/분석 파이프라인

  • 1) 프레임 추출 — ffmpeg로 영상에서 일정 간격으로 대표 프레임 추출
  • 2) 멀티레이블 태깅 모델
    • EfficientNet-B0를 백본으로 사용, ImageNet 사전학습 가중치 활용
    • 기존 classifier의 마지막 Linear 레이어를 Identity로 교체하고, num_labels 크기의 Linear 헤드(tag_head)만 새로 학습
    • 한 영상에 여러 태그가 동시에 붙을 수 있으므로 Softmax 대신 Sigmoid를 사용해 태그별 독립 확률을 얻음
    • 확률이 일정 기준(예: 0.3~0.5) 이상인 태그만 선택해 DB에 저장
  • 3) 태그 정규화 — 예: sea, ocean을 모두 화면에서는 “바다”로 통합 표시
  • 4) 검색/추천에 활용 — 태그 + 제목 + 설명 조합으로 기본 검색 구현, 이후 벡터 검색과 결합 예정
  • 5) WhisperX + LLM(설계) — 긴 영상은 WhisperX로 전사 후, 요약/챕터/FAQ를 LLM으로 생성하는 파이프라인 설계 및 프롬프트 실험 진행

보안 & 운영 관점에서의 설계

  • 네트워크 분리 — 앱 서버/DB 서버를 서로 다른 노트북으로 구성, DB는 사설망(10.10.10.x)에서만 접근 가능
  • iptables 정책 — DB 서버는 앱 서버 IP 이외 연결 차단, 외부에서 직접 DB 포트 접근 불가
  • Spring Security + JWT — 비로그인 사용자는 업로드/좋아요/찾기 등 주요 API에 접근 불가
  • CORS 정책 — 프런트 도메인 허용, /admin API는 별도로 제어
  • SSL/TLS — Nginx + Certbot으로 aicollector.co.kr에 HTTPS 적용, HSTS/리다이렉트 설정
  • 운영 자동화 — Docker 컨테이너 재시작 스크립트, 로그 로테이션, 디스크 용량 모니터링 구조 설계

회고 (배운 점)

  • 단순한 “영상 목록 사이트”가 아니라, AI 태깅/요약/검색 파이프라인 설계가 서비스 가치의 핵심이라는 점
  • 대용량 영상 업로드에서는 보안(CORS/인증)과 스트리밍 성능을 함께 고려해야 한다는 점
  • Oracle DB를 외부 서버로 분리하면서 네트워크/방화벽/SSL까지 직접 다뤄본 경험
  • 모델 자체 성능보다, 태그 정규화·임계값 튜닝·UX가 사용자 만족에 더 큰 영향을 미칠 수 있다는 점

회고 (개선 방향)

  • Finding 페이지에 벡터 검색 + BM25 하이브리드 검색 실제 적용
  • WhisperX 및 LLM을 통해 요약/챕터/FAQ 자동 생성까지 파이프라인 완성
  • 영상 유해성 판별 모델을 강화해 관리자 부담 감소
  • 모바일 사용자를 위한 요약 카드/스와이프 UX 최적화
